# F151 Exact RED/GREEN/REFACTOR Slice Manifest

本manifest在实现前冻结行为切片、nodeid、层级、唯一RED oracle与production ownership。exact machine-readable production path/glob/inventory/symbol scopes在[`rgr-slice-scopes.v1.json`](rgr-slice-scopes.v1.json)；本表最后一栏仅为人读摘要，不可覆盖machine scope。除明确标为`CHARACTERIZATION`或`ATOMIC`的条目，每个slice必须按同一组`test_nodeids`连续执行RED→GREEN→REFACTOR；collection/import/path/venv error、skip、rerun或换selector均无效。任一changed behavior production/test path未映射、overlap无symbol partition、free-text scope或0 required slice但changed set非0均失败。

## Exact command encoding

Python slice的每个phase命令由以下固定argv编码；这不是调用者shell变量，runner必须按字段逐字节构造并把实际完整env/argv写入lifecycle exact `invocation.json`。T048前使用pre-SDK lock（含`packages/sdk/src`），T048完成后使用post-SDK lock（删除该分量）；不存在路径虽对Python无害，也不得用来掩盖phase状态。C23/C24固定使用post-SDK lock。

```text
env PYTHONNOUSERSITE=1 PYTHONPATH="$PWD/octoagent/packages/core/src:$PWD/octoagent/packages/provider/src:$PWD/octoagent/packages/protocol/src:$PWD/octoagent/packages/tooling/src:$PWD/octoagent/packages/skills/src:$PWD/octoagent/packages/policy/src:$PWD/octoagent/packages/memory/src:$PWD/octoagent/packages/sdk/src:$PWD/octoagent/apps/gateway/src" uv run --project octoagent --no-sync python -m pytest -q -rA <本表test_nodeids按空格原序展开> --junitxml=.specify/features/151-runtime-boundary-architecture-truth/evidence/local/runs/<slice_id>/<RED|GREEN|REFACTOR>/junit.xml
```

post-SDK唯一差异是`PYTHONPATH`删除`$PWD/octoagent/packages/sdk/src`，其余argv字节不变；runner按slice task和retirement state选定，不接受调用者临时覆盖。

上面的pytest argv是runner记录到`invocation.json`的inner argv，不允许调用者直接指定输出root。每行三phase仅phase值不同，selector字节完全相同；runner拒绝任何额外`--reruns`/selector，并在`evidence/local/runs/<slice_id>/<phase>/`精确生成`junit.xml/stdout.txt/stderr.txt/exit-code.txt/invocation.json/tree.json`六件套。`.bin`、`run.json`、非canonical root、缺件或多件全部失败。RED期望exit=1、errors=0且仅目标assertion failure；GREEN/REFACTOR期望exit=0且selected node全部pass。

Frontend slice使用以下固定encoding，三phase同样只替换目录名：

```text
(cd octoagent/frontend && npm exec vitest -- run <file> -t "<test name>" --reporter=default --reporter=junit --outputFile.junit=../../.specify/features/151-runtime-boundary-architecture-truth/evidence/local/runs/<slice_id>/<RED|GREEN|REFACTOR>/junit.xml)
```

formal runner唯一语法是`check-runtime-architecture.py tdd-evidence run --slice <id> --phase <RED|GREEN|REFACTOR> --mode local-working-tree --base-ref <ref> --evidence-index .specify/features/151-runtime-boundary-architecture-truth/evidence/evidence-index.v2.json`；verify唯一语法是`... tdd-evidence verify --mode <local-working-tree|committed> --base-ref <ref> --evidence-index .specify/features/151-runtime-boundary-architecture-truth/evidence/evidence-index.v2.json [--through-task <id>]`。runner从manifest选择Python或Frontend inner argv并派生canonical目录，不接受output参数或环境覆盖；`verify`重算六件套hash、解析JUnit并与索引交叉核验。JSONL或raw文本自述不能覆盖JUnit事实。
