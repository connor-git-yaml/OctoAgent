# Acceptance Matrix - Feature 050 Agent Management Simplification

## 1. 主路径矩阵

| 场景 | 预期结果 | 建议验证方式 |
|------|---------|-------------|
| 打开 `Agents` 页面 | 首屏展示当前项目主 Agent + 当前项目已创建 Agent 列表 | RTL 页面渲染测试 |
| 当前项目无普通 Agent | 页面显示空状态和“新建 Agent”按钮 | RTL 空状态测试 |
| 点击“新建 Agent” | 打开模板选择流，而不是直接进入技术编辑工作台 | 交互测试 |
| 从模板创建 Agent | 成功生成新 Agent 并返回列表 | 页面集成测试 |
| 编辑主 Agent | 可保存，但无删除按钮 | 页面集成测试 |
| 编辑普通 Agent | 可保存，存在删除按钮 | 页面集成测试 |
| 切换项目 | 列表刷新为目标项目的 Agent | 页面集成测试 |

## 2. 表单矩阵

| 字段 | 预期表达 | 反例（不得出现） |
|------|---------|------------------|
| 使用的 LLM | 单选 / 下拉 | 纯文本输入模型别名 |
| 默认工具组 | 多选控件 | 让用户手输每行一个 tool group |
| 固定工具 | 搜索多选 | 让用户手输 raw tool name |
| 所属项目 | 单选 / 下拉 | 隐式沿用而不展示 |
| 能力绑定 | 分组多选 | 只显示技术术语“Provider 白名单” |

## 3. 迁移矩阵

| 场景 | 预期结果 |
|------|---------|
| 当前项目默认 Agent 仍是 builtin | 页面提示建立项目主 Agent，迁移后可正常编辑 |
| 删除普通 Agent | 需要确认；主 Agent 绑定不受影响 |
| 模板更新 | 已创建 Agent 不被静默覆盖 |

## 4. 术语矩阵

默认主路径应使用：

- 主 Agent
- 已创建 Agent
- 模板
- 所属项目
- 使用的模型
- 默认工具组
- 固定工具
- 能力绑定

默认主路径不应直接使用：

- Worker Profile
- archetype
- runtime kinds
- policy refs
- tool profile
- Provider 白名单
