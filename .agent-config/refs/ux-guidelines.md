# Web UI / UX 详细规范

## 核心原则

Web 端页面默认面向**普通非技术用户**设计，优先降低理解成本和操作门槛。

## 具体规则

- 页面首要回答：**当前发生了什么、对我有什么影响、下一步该做什么**
- 主界面避免暴露 debug / 运维术语与原始技术字段（内部 ID、scope 原始标识、调试状态码、索引细节、flush/replay、原始异常栈等）
- 技术信息必须放到 **Advanced / 管理台 / 诊断区 / 折叠区**，不得占据普通用户主路径
- 状态/报错/提醒文案先解释影响，再给出明确动作；不要只抛出底层状态名
- 配置引导坚持**最小必要原则**：只告诉用户"最少需要配置什么"和"去哪里配置"
- 新增/重构页面检查是否有"只有开发者能看懂"的区块；若有，改写或迁移到高级入口
