## Heartbeat vs Cron 分工

**Heartbeat（本文件）** 是 Worker 任务内自检机制——在执行长任务过程中定期暂停检查自身状态。**Cron 定时任务** 是独立调度的周期性工作，与当前 Worker 派发无关。Worker 的 HEARTBEAT 自检结果通过当前 Worker 回报通道送给主 Agent（运行时支持时使用 A2A 状态字段），**不直接面向用户**。

## 自检触发条件

在长任务执行过程中，以下时机应暂停执行并自检：

- 连续工作超过 5 个工具调用后
- 遇到非预期错误或工具调用连续失败时
- 任务方向发生偏转、需要重新评估路径时
- 花费时间明显超出对任务复杂度的预期时
- 发现 objective 假设不成立或需越权操作时（**立即** escalate）

## 自检清单

1. **进度** — 已完成哪些步骤？离 objective 还有多远？
2. **方向** — 工作是否仍围绕主 Agent 派发的 objective？有无偏离？
3. **工具** — 当前工具是否合适？有更高效的替代吗？
4. **收口** — objective 是否已达成？是否在过度探索？
5. **阻碍** — 是否遇到无法独立解决的问题？是否需要 escalate？
6. **越界** — 是否有动作触及 escalation policy（不可逆操作 / 越权变更 / 假设失效）？

## 进度回报格式（对象：主 Agent，通过当前 Worker 回报通道）

向主 Agent 回报时按此结构组织（**不直接面向用户**）：

- **已完成**：做了什么，产出了哪些可验证的成果
- **阻碍**：遇到什么问题，为什么卡住，是否需要主 Agent 决策
- **下一步**：打算做什么，预计还需要多久，或回报"等待主 Agent 指示"

## 收口标准（回报触发）

满足以下任一条件时主动结束当前工作并通过 Worker 回报通道（运行时支持时填充 A2A 状态）回报主 Agent：

- objective 已达成，验收条件已满足
- 继续探索不会产生有意义的增量价值
- 遇到无法自行解决的阻碍，需主 Agent 决策（**不直接打扰用户**）
- 任务假设不成立，需主 Agent 与用户对齐后再调整 objective
- 检测到越界 / 不可逆操作 / 安全边界事件 — **立即** escalate
