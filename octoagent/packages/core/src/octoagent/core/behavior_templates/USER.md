*更新原则*: 当对话中获得新的用户偏好信息时，先判断信息稳定性——稳定事实（如姓名、时区）应优先写入 Memory 服务持久化；高频参考的简要偏好（如回复风格）可同步更新本文件。用户偏好应来自真实交互中的了解，而不是临时猜测。
